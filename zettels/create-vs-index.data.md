# Create vs Index

[[shard-write.data]] always uses INDEX (upsert) semantics (`autoGeneratedTimestamp = -1`).

- CREATE semantics (fail if document exists) requires error handling — the write must return a `Result` indicating success or conflict
- Blocked on sum types / [[result-types.types]]
- Future: `Shard.write` takes a mode parameter or two separate builtins (`Shard.create` vs `Shard.upsert`)

**Depends on**: [[shard-write.data]], [[result-types.types]]
**Enables**: (none directly)
**Connections**:
- part-of: [[primary-shard-write.data]] — INDEX vs CREATE is a parameter to `applyIndexOperationOnPrimary`
- constrains: [[index-bulk.data]] — Index.bulk also faces INDEX vs CREATE semantics
