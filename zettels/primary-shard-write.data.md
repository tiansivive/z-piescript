# Primary Shard Write

[[shard-write.data|Shard.write]] bypasses transport -- calls `IndexShard.applyIndexOperationOnPrimary()` directly on the local primary.

- No replication, no ingest pipelines, no routing.
- INDEX (upsert) semantics via `autoGeneratedTimestamp = -1`.
- Maximum performance, maximum responsibility.
- The user controls replication monitoring via `Shard.globalCheckpoint`.

**Depends on**: [[shard-write.data]]
**Enables**: (none directly)
**Connections**:
- contrasts-with: deliberately bypasses the `TransportShardBulkAction` path that handles replication/ingest
- motivates: [[session-types.types]] — future linearity (Phase 6) could enforce write->replicate protocol via session types
- complements: [[replication-model.data]] — replicas catch up after primary write via translog
- tradeoff-with: [[create-vs-index.data]] — INDEX vs CREATE is a parameter to applyIndexOperationOnPrimary
